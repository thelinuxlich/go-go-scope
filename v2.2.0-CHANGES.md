# go-go-scope v2.2.0 Changes Summary

This document summarizes all the changes made for version 2.2.0 of go-go-scope.

## ✅ Completed Features

### 1. Cache/Memoization with Persistence Adapter Support

**New Files:**
- `packages/go-go-scope/src/cache.ts` - In-memory cache implementation
- `packages/persistence-redis/src/cache.ts` - Redis cache adapter
- `packages/persistence-postgres/src/cache.ts` - PostgreSQL cache adapter
- `packages/persistence-mysql/src/cache.ts` - MySQL cache adapter
- `packages/persistence-sqlite/src/cache.ts` - SQLite cache adapter

**Modified Files:**
- `packages/go-go-scope/src/persistence/types.ts` - Added `CacheProvider` and `CacheStats` interfaces
- `packages/go-go-scope/src/index.ts` - Exported cache utilities
- All persistence adapters updated to implement `CacheProvider`

**Features:**
- In-memory cache with TTL and LRU eviction
- Distributed cache support via all persistence adapters
- Cache statistics (hits, misses, hit ratio)
- Pattern-based key matching

### 2. Dead Letter Queue (DLQ) for Scheduler

**Modified Files:**
- `packages/scheduler/src/types.ts` - Added DLQ types (already existed)
- `packages/scheduler/src/scheduler.ts` - Added DLQ methods:
  - `addToDLQ()` - Move failed jobs to DLQ
  - `getDLQJobs()` - Query DLQ jobs
  - `replayFromDLQ()` - Replay a job from DLQ
  - `purgeDLQ()` - Clean up DLQ jobs

**Features:**
- Automatic job archival after max retries exceeded
- Configurable DLQ storage (separate from main storage)
- `onDeadLetter` callback for custom handling
- `jobDeadLettered` event

### 3. Backpressure Strategies for Channels

**Modified Files:**
- `packages/go-go-scope/src/types.ts` - Added `BackpressureStrategy` type and `ChannelOptions`
- `packages/go-go-scope/src/channel.ts` - Implemented all strategies
- `packages/go-go-scope/src/errors.ts` - Added `ChannelFullError`
- `packages/go-go-scope/src/index.ts` - Exported new types

**Strategies Implemented:**
- `block` - Wait until space available (default)
- `drop-oldest` - Remove oldest item to make room
- `drop-latest` - Drop the new item when buffer full
- `error` - Throw `ChannelFullError` when buffer full
- `sample` - Time-based sampling with sliding window

### 4. Idempotency as Task Option

**New Files:**
- `packages/go-go-scope/src/idempotency.ts` - In-memory idempotency provider

**Modified Files:**
- `packages/go-go-scope/src/persistence/types.ts` - Added `IdempotencyProvider` interface
- `packages/go-go-scope/src/types.ts` - Added `idempotencyKey` and `idempotencyTTL` to `TaskOptions`
- `packages/go-go-scope/src/scope.ts` - Implemented idempotency logic in task execution
- `packages/go-go-scope/src/index.ts` - Exported idempotency utilities

**Features:**
- String or function-based idempotency keys
- Configurable TTL per task
- Errors are NOT cached (tasks will retry on failure)
- Inheritance from parent scopes

### 5. Enhanced Stream Operations

**Modified Files:**
- `packages/stream/src/index.ts` - Added new operations

**New Operations:**
- `collect(pf)` - Partial function collect
- `collectWhile(pf)` - Collect while partial function returns defined values
- `grouped(size)` - Alias for buffer with better semantics
- `pipe(...fns)` - Functional composition helper
- `runHead()` - Effect-style alias for `first()`
- `runLast()` - Effect-style alias for `last()`
- `runSum()` - Effect-style alias for `sum()`

## 📝 Migration Guide

### For Cache Usage
```typescript
// Before: Manual caching
const cache = new Map()

// After: Using CacheProvider
import { InMemoryCache } from 'go-go-scope'
const cache = new InMemoryCache({ maxSize: 1000 })

// Or with persistence
await using s = scope({
  persistence: { cache: redisAdapter }
})
```

### For Idempotency
```typescript
// Before: Manual deduplication

// After: Built-in idempotency
const [err, result] = await s.task(
  () => chargeCustomer(id),
  { idempotencyKey: `charge:${id}`, idempotencyTTL: 60000 }
)
```

### For Typed Schedules
```typescript
// Before: Untyped
const scheduler = new Scheduler({ storage })
scheduler.onSchedule('send-email', (job) => {
  // job.payload is any
})

// After: Typed
type AppSchedules = {
  'send-email': { to: string; subject: string }
}
const scheduler = new Scheduler<AppSchedules>({ storage })
scheduler.onSchedule('send-email', (job) => {
  // job.payload is fully typed!
  const { to, subject } = job.payload
})
```
