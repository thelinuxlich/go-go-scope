Structured Concurrency
for TypeScript

Write concurrent code with automatic cleanup and cancellation propagation using the using and await using syntax.

Get Started GitHub

Installation

Install go-go-scope using your favorite package manager:

npm install go-go-scope
Requirements

TypeScript 5.2+ with "target": "ES2022" or higher. Requires native support for Symbol.dispose and Symbol.asyncDispose (Node.js 24+ or Bun 1.2+).

Quick Start

Your First Example

The simplest way to understand go-go-scope is with a timeout example:

TypeScript 
import { scope } from 'go-go-scope'

// Fetch data with a 5-second timeout
async function fetchData() {
  await using s = scope({ timeout: 5000 })
  
  const [err, data] = await s.task(async ({ signal }) => {
    const response = await fetch('/api/data', { signal })
    return response.json()
  })
  
  if (err) {
    console.log('Failed:', err.message)
    return null
  }
  
  return data
}
What just happened?
  • scope({ timeout: 5000 }) creates a scope that auto-cancels after 5 seconds
  • s.task() creates a task that can be cancelled
  • { signal } is an AbortSignal you pass to fetch() for cancellation
  • [err, data] is a Result tuple - always [error, value]
  • await using automatically cleans up when done

Key Pattern: The Result Tuple

Every task returns [error, value]:

TypeScript 
const [err, user] = await s.task(() => fetchUser(1))

if (err) {
  // Handle error
} else {
  // Use user
}

This is different from Promises that throw errors. Result tuples never throw!

Structured Concurrency

Structured concurrency is a programming paradigm that ensures:

  1. No task is left behind - When a scope ends, all tasks inside it are cancelled
  2. No fire-and-forget - Every async operation has a parent that waits for it
  3. Cleanup is guaranteed - Resources are always cleaned up, even on error
TypeScript 
// Regular function - structured!
function parent() {
  const result = child()  // Parent waits for child
  console.log(result)     // This always runs after child
}

// Unstructured async (the problem)
async function parent() {
  child()  // Fire and forget! Parent doesn't wait
  console.log('Done')  // This might run before child finishes
}

// Structured async with go-go-scope
async function parent() {
  await using s = scope()
  const [err, result] = await s.task(() => child())  // Parent waits
  console.log(result)  // This always runs after child
}

The Scope

A Scope is a container for async operations. Think of it as a "workspace" for your tasks.

Creating a Scope

TypeScript 
import { scope } from 'go-go-scope'

// Simple scope
await using s = scope()

// With timeout
await using s = scope({ timeout: 5000 })

// With OpenTelemetry tracing
await using s = scope({
  name: 'my-operation',
  tracer: trace.getTracer('my-app')
})

// With concurrency limit
await using s = scope({ concurrency: 3 })

// With circuit breaker
await using s = scope({
  circuitBreaker: {
    failureThreshold: 5,
    resetTimeout: 30000
  }
})

// Combined
await using s = scope({
  name: 'complex-operation',
  timeout: 30000,
  concurrency: 5,
  tracer: trace.getTracer('my-app'),
  circuitBreaker: { failureThreshold: 3 }
})

Resource Cleanup

Resources are cleaned up in LIFO order (reverse of creation):

TypeScript 
async function processFile(filePath: string) {
  await using s = scope()
    .provide('tempDir', () => createTempDir(), (dir) => dir.cleanup())
    .provide('fileHandle', () => openFile(filePath), (fh) => fh.close())
    .provide('db', () => openDatabase(), (db) => db.close())
  
  // Use the resources
  const data = await s.use('fileHandle').read()
  
  // Cleanup order (automatic):
  // 1. Database connection closes
  // 2. File handle closes
  // 3. Temp directory is cleaned up
}

The Task

A Task is a unit of work within a scope. It's like a Promise, but:

  • Lazy - Doesn't start until you await it
  • Cancellable - Respects the AbortSignal
  • Returns a Result - Never throws, always returns [error, value]

Lazy Execution

TypeScript 
await using s = scope()

const primary = s.task(() => fetchFromPrimary())
const fallback = s.task(() => fetchFromFallback())

// Try primary first
const [err, result] = await primary
if (err) {
  // Only now does fallback start!
  const [fallbackErr, fallbackResult] = await fallback
  return fallbackResult
}
return result

Parallel Execution

TypeScript 
await using s = scope({ concurrency: 3 })

const result = await s.parallel([
  ({ signal }) => fetchUser(1, { signal }),
  ({ signal }) => fetchUser(2, { signal }),
  ({ signal }) => fetchUser(3, { signal }),
])

// Access successful results
for (const { index, value } of result.completed) {
  console.log(`Task ${index}:`, value)
}

// Access errors
for (const { index, error } of result.errors) {
  console.log(`Task ${index} failed:`, error)
}

Result Tuples

Every task returns a Result tuple: [error, value]

Why Result Tuples?

Promises throw errors:

TypeScript 
try {
  const user = await fetchUser(1)  // Might throw!
} catch (err) {
  // Handle error
}

Tasks return errors:

TypeScript 
const [err, user] = await s.task(() => fetchUser(1))
if (err) {
  // Handle error
} else {
  // Use user
}
Benefits
  • Explicit - You must check for errors
  • Type-safe - TypeScript knows both types
  • Composable - Easy to aggregate results
  • No try/catch - Cleaner control flow

Cancellation

Every task receives an AbortSignal that you can use for cancellation:

TypeScript 
await s.task(async ({ signal, logger }) => {
  // Pass to fetch for automatic cancellation
  const response = await fetch(url, { signal })
  
  // Or check manually
  if (signal.aborted) {
    logger.warn('Task was cancelled')
    throw new Error('Cancelled!')
  }
  
  // Or listen for abort events
  signal.addEventListener('abort', () => {
    logger.info('Task was cancelled')
  })
})

When is Cancellation Triggered?

  1. Timeout - Scope timeout is reached
  2. Scope disposal - await using block ends
  3. Parent cancellation - Parent scope is cancelled
  4. Manual abort - You call scope.signal.abort()

Channels

Channels provide Go-style concurrent communication between tasks with configurable backpressure strategies.

TypeScript 
await using s = scope()
const ch = s.channel<string>(100)  // Buffer capacity of 100

// Producer
s.task(async () => {
  for (const log of logs) {
    await ch.send(log)  // Blocks if buffer full
  }
  ch.close()
})

// Consumer with native async iteration
for await (const log of ch) {
  await processLog(log)
}

Backpressure Strategies

Strategy Behavior Use Case
block (default) Wait until space available Reliable delivery, bounded memory
drop-oldest Remove oldest item to make room Keep recent data (telemetry)
drop-latest Drop the new item Prioritize processing over ingestion
error Throw ChannelFullError Explicit backpressure handling

Error Handling

Use go-go-try alongside go-go-scope for typed error handling:

TypeScript 
import { scope } from 'go-go-scope'
import { taggedError, success, failure } from 'go-go-try'

const DatabaseError = taggedError('DatabaseError')
const NetworkError = taggedError('NetworkError')

async function fetchUser(id: string) {
  await using s = scope()
  
  // Database operation - errors wrapped in DatabaseError
  const [dbErr, user] = await s.task(
    () => queryDatabase(id),
    { errorClass: DatabaseError }
  )
  if (dbErr) return failure(dbErr)
  
  // Network operation - errors wrapped in NetworkError
  const [netErr, enriched] = await s.task(
    () => enrichUserData(user!),
    { errorClass: NetworkError }
  )
  if (netErr) return failure(netErr)
  
  return success(enriched)
}

Concurrency Patterns

go-go-scope provides powerful primitives for concurrent programming. Here's how to use them effectively.

Worker Threads

Offload CPU-intensive tasks to worker threads to avoid blocking the main thread:

TypeScript 
import { scope } from 'go-go-scope'

// CPU-intensive calculation
function fibonacci(n: number): number {
  if (n < 2) return n
  return fibonacci(n - 1) + fibonacci(n - 2)
}

async function parallelProcessing() {
  await using s = scope()

  // Run CPU-intensive tasks in parallel using workers
  const [err, results] = await s.parallel([
    () => fibonacci(40),
    () => fibonacci(39),
    () => fibonacci(38),
  ], { 
    workers: true,  // Use worker threads
    concurrency: 2  // Max 2 workers at a time
  })

  if (err) {
    console.error('Some tasks failed:', err)
    return
  }

  console.log('Fibonacci results:', results)
}

Semaphores

Control access to limited resources with semaphores:

TypeScript 
import { scope } from 'go-go-scope'

async function rateLimitedRequests(urls: string[]) {
  await using s = scope()

  // Create a semaphore with 5 permits
  const semaphore = s.semaphore(5)

  const results = await Promise.all(
    urls.map(async (url) => {
      // Acquire a permit (blocks if none available)
      await semaphore.acquire()
      
      try {
        const response = await fetch(url)
        return response.json()
      } finally {
        // Always release the permit
        semaphore.release()
      }
    })
  )

  return results
}

Select Statement

Wait on multiple channel operations simultaneously, similar to Go's select:

TypeScript 
import { scope } from 'go-go-scope'

async function multiplexChannels() {
  await using s = scope()

  const ch1 = s.channel()
  const ch2 = s.channel()
  const timeout = s.channel()

  // Producer 1
  s.task(async () => {
    await ch1.send('message from ch1')
  })

  // Producer 2
  s.task(async () => {
    await ch2.send('message from ch2')
  })

  // Timeout
  s.task(async () => {
    await s.delay(5000)
    timeout.close()
  })

  // Select - wait for first ready channel
  const result = await s.select([
    { channel: ch1, action: (msg) => console.log('Got:', msg) },
    { channel: ch2, action: (msg) => console.log('Got:', msg) },
    { channel: timeout, action: () => console.log('Timeout!') }
  ])

  return result
}

Broadcast Channels

Send messages to multiple consumers simultaneously:

TypeScript 
import { scope } from 'go-go-scope'

async function eventBroadcasting() {
  await using s = scope()

  // Create broadcast channel - all subscribers receive every message
  const broadcast = s.broadcast()

  // Subscribe multiple consumers
  const consumer1 = broadcast.subscribe()
  const consumer2 = broadcast.subscribe()
  const consumer3 = broadcast.subscribe()

  // Publish messages
  await broadcast.send('Event 1')
  await broadcast.send('Event 2')

  // All consumers receive both events
  for await (const event of consumer1) {
    console.log('Consumer 1:', event)
  }
}

Resilience Patterns

Build fault-tolerant applications with built-in resilience patterns.

Circuit Breaker

Prevent cascading failures by automatically detecting when a service is unhealthy:

TypeScript 
import { scope } from 'go-go-scope'

async function resilientServiceCall() {
  await using s = scope({
    circuitBreaker: {
      failureThreshold: 5,     // Open after 5 failures
      resetTimeout: 30000,     // Try again after 30s
      successThreshold: 2,     // Need 2 successes to close
      onStateChange: (from, to) => {
        console.log(`Circuit: ${from} → ${to}`)
      },
      onOpen: (failures) => {
        alert(`Service down after ${failures} failures`)
      }
    }
  })

  const [err, result] = await s.task(() => callExternalAPI())

  if (err) {
    if (err.message === 'Circuit breaker is open') {
      // Use cached data or fallback
      return getCachedData()
    }
    throw err
  }

  return result
}

Timeout Patterns

Set timeouts at different levels for granular control:

TypeScript 
import { scope } from 'go-go-scope'

async function granularTimeouts() {
  // Overall operation timeout
  await using s = scope({ timeout: 30000 })

  // Quick health check with short timeout
  const [healthErr, healthy] = await s.task(
    () => checkHealth(),
    { timeout: 5000 }
  )

  if (healthErr || !healthy) {
    throw new Error('Service unhealthy')
  }

  // Main operation with medium timeout
  const [err, result] = await s.task(
    () => processData(),
    { timeout: 20000 }
  )

  return result
}

Graceful Shutdown

Handle process termination gracefully with configurable strategies:

TypeScript 
import { scope } from 'go-go-scope'

async function main() {
  await using s = scope()

  // Setup graceful shutdown
  const shutdown = setupGracefulShutdown(s, {
    signals: ['SIGTERM', 'SIGINT'],
    timeout: 30000,
    beforeShutdown: async () => {
      console.log('Starting graceful shutdown...')
      await closeDatabaseConnections()
      await flushLogs()
    },
    afterShutdown: () => {
      console.log('Shutdown complete')
    }
  })

  // Your application logic
  await startServer()
}

// When process receives SIGTERM:
// 1. Stop accepting new requests
// 2. Wait for in-flight requests (up to 30s)
// 3. Clean up resources
// 4. Exit cleanly

Deadlock Detection

Detect and prevent deadlocks in concurrent code:

TypeScript 
import { scope } from 'go-go-scope'

async function safeLockAcquisition() {
  await using s = scope({
    deadlockDetection: {
      enabled: true,
      timeout: 10000,  // Warn if lock held > 10s
      onPotentialDeadlock: (info) => {
        console.warn('Potential deadlock detected:', info)
      }
    }
  })

  // Acquire multiple locks safely
  const lock1 = await s.acquireLock('resource-1')
  const lock2 = await s.acquireLock('resource-2')

  // Use resources...

  // Automatic release on scope disposal
}

Guide: HTTP Client with Retry

A resilient HTTP client with automatic retries and exponential backoff:

TypeScript 
import { scope, exponentialBackoff } from 'go-go-scope'

function createHttpClient(baseUrl: string) {
  return {
    async get<T>(url: string): Promise<T> {
      await using s = scope()

      const [err, result] = await s.task(
        async ({ signal }) => {
          const response = await fetch(`${baseUrl}${url}`, { signal })
          if (!response.ok) throw new Error(`HTTP ${response.status}`)
          return response.json() as Promise<T>
        },
        {
          retry: {
            max: 3,
            delay: exponentialBackoff({ initial: 100, max: 5000, jitter: 0.3 })
          }
        }
      )

      if (err) throw err
      return result!
    }
  }
}

Guide: Database Connection Pool

Manage a pool of database connections with automatic cleanup:

TypeScript 
import { scope } from 'go-go-scope'

async function withDatabase<T>(
  connectionString: string,
  fn: (db: DatabaseConnection) => Promise<T>
): Promise<T> {
  await using s = scope()

  const pool = s.resourcePool<DatabaseConnection>({
    create: async () => createConnection(connectionString),
    destroy: async (conn) => conn.close(),
    min: 2,
    max: 10,
    acquireTimeout: 5000
  })

  const conn = await pool.acquire()
  try {
    return await fn(conn)
  } finally {
    await pool.release(conn)
  }
}

Guide: Circuit Breaker Pattern

Protect services with circuit breaker pattern:

TypeScript 
import { scope } from 'go-go-scope'

async function callExternalService() {
  await using s = scope({
    circuitBreaker: {
      failureThreshold: 5,    // Open after 5 failures
      resetTimeout: 30000,    // Try again after 30 seconds
      onStateChange: (from, to, count) => {
        console.log(`Circuit: ${from} -> ${to}`)
      },
      onOpen: (count) => {
        alertService.notify(`Circuit opened after ${count} failures`)
      }
    }
  })

  const [err, result] = await s.task(() => fetchExternalData())

  if (err) {
    if (err.message === 'Circuit breaker is open') {
      return getCachedData()  // Use fallback
    }
    throw err
  }

  return result
}

Guide: Retry Patterns

Exponential Backoff

TypeScript 
import { exponentialBackoff } from 'go-go-scope'

const [err, result] = await s.task(
  () => fetchData(),
  {
    retry: {
      max: 5,
      delay: exponentialBackoff({ initial: 1000, max: 30000, jitter: 0.3 })
      // Attempt 1: ~1000ms (±30% jitter)
      // Attempt 2: ~2000ms (±30% jitter)
      // Attempt 3: ~4000ms (±30% jitter)
    }
  }
)

Conditional Retry

TypeScript 
const [err, result] = await s.task(
  () => fetchData(),
  {
    retry: {
      max: 3,
      retryCondition: (error) => {
        // Only retry network errors
        return error instanceof NetworkError
      }
    }
  }
)
