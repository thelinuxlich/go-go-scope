### 📊 Statistical Confidence

- **Warmup iterations:** 100 (ensures JIT optimization)
- **Measurement samples:** 50 per benchmark
- **Outlier filtering:** IQR method (removes extreme values)
- **Cooldown between suites:** 2 seconds (prevents thermal throttling)
- **Coefficient of variation:** < 10% for most benchmarks (highly consistent)

### 🎯 Recommendations

1. **Use Bun** for:
   - High-throughput scope creation/destruction
   - Channel-heavy applications
   - Microservices with frequent context switching

2. **Use Node.js** for:
   - I/O-bound applications (APIs, web servers)
   - Production workloads requiring stability
   - Ecosystem compatibility

3. **Use Deno** for:
   - Stream processing pipelines
   - Security-focused applications
   - TypeScript-first projects

### ⚠️ Notes

- Benchmarks run in isolated mode with cooldown periods
- Results may vary based on hardware, OS, and system load
- For production decisions, run benchmarks on your target infrastructure
- All runtimes show excellent performance (> 1M ops/sec for simple operations)
